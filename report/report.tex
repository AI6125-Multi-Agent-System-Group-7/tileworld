\begin{abstract}
This report presents Group 7's six-agent system for Tileworld, a partially observable and dynamic grid environment with fuel, sensing, and carrying constraints.
Our implementation combines a shared engineering layer (\texttt{Group7AgentBase}), protocol-driven collaboration (G7P2), and agent-specific action policies.
We evaluate the system on three parameter sets and report mean reward over repeated runs.
Results and ablations show that both communication and policy specialization are necessary for strong team performance, especially in larger and more dynamic maps.
\end{abstract}


\section{Introduction \& Background}

Tileworld is a standard benchmark for studying multi-agent decision making under uncertainty \cite{intro2tileworld,hist_tileworld}. Agents operate on a grid with partial observability, stochastic object dynamics, limited fuel, and finite carrying capacity; hence effective behavior requires both local intelligence and team coordination.

In this project, six agents jointly act in the same environment using a Sense$\rightarrow$Communicate$\rightarrow$Plan$\rightarrow$Act loop. We focus on three design questions: (i) how to share local observations efficiently, (ii) how to reduce conflict and duplicated effort, and (iii) how much performance comes from shared infrastructure versus agent-specific policies. The rest of this report presents our implementation, experimental settings, quantitative results, and ablation findings.

\section{Environment \& Experiment Setup}

We ran experiments in a tileworld simulator implemented in Java 1.8 with the Java3D v1.5.1 and MASON 14 package. One simulation will run for 5000 steps and we take the final reward as the metric of the agents system; 
different simulations will start with a randomly generated seed. 
The reward is defined by the number of holes filled. The initial fuel is fixed at the maximum fuel carriage of 500.

We evaluated 3 different configurations that have different world sizes, object lifetimes, and object creation distributions. \\

\section{Agent Design}

\subsection{Group7AgentBase - Abstract Base Class}

Group7AgentBase is the shared control and coordination layer for all Group 7 agents.
It standardizes low-level navigation (A*-based movement, path reuse, obstacle-aware stepping, and zig-zag fallback exploration), so each agent can focus on high-level policy.
It also unifies fuel-safe behavior through common refuel handling.
On collaboration, the base implements G7P2 messaging, including directed and broadcast communication, and regularly publishes core shared signals such as discoveries and local sensor snapshots.
This design isolates reusable infrastructure from agent-specific decision logic and provides a consistent interface for comparing policies.

\subsection{AgentHanny - The Proactive Learner}

AgentHanny updates a shared world estimate from local and team observations, then plans by maximizing expected utility.

\paragraph{Planning}
Actions are selected by \textbf{utility over plan}, where each plan is a sequence of pickup/fill decisions from the current state.
The key signal is expected utility, computed using survival-based estimates of whether candidate objects remain available at arrival time (derivation in the appendix \cite{kleinbaum2012survival}).
The agent executes the highest-EU plan and locks only the currently executed target to reduce conflict.
The overall control loop is shown in Figure~\ref{fig:hanny-flow}.

\paragraph{Communications}
AgentHanny follows G7P2 and integrates incoming team messages.
Target locks are renewed while a target remains valid and are released immediately after completion, invalidation, pre-emption, or mode switching (e.g., replanning/refuel).

\paragraph{Memory Layer}
To support online estimation, the agent keeps augmented memory metadata (e.g., object type and temporal observation cues) synchronized with the shared world model.

\paragraph{Exploration}
For team-level coverage, the map is partitioned into a dynamic \(n \times n\) grid (\(n=WorldSize//15\)).
Sections are scored by teammate density, explored coverage, and travel distance; far infeasible sections are filtered.
The best-scoring section is explored first, with zig-zag fallback when no valid section exists.

\subsection{AgentEnge - The Priority-Driven Planner}

AgentEnge follows a strict four-level priority planner that combines fuel safety, task execution, and structured exploration with team conflict avoidance.

\paragraph{Planning}
Decisions are evaluated every cycle through a fixed cascade:
\begin{enumerate}
  \item \textbf{Survival} (\(fuel<180\)): go to fuel station immediately.
  \item \textbf{Fill} (carrying tile and valid hole exists): pick the nearest feasible unlocked hole.
  \item \textbf{Pickup} (capacity available and tile available): lock and collect nearest feasible tile.
  \item \textbf{Exploration}: sweep the assigned area.
\end{enumerate}
A* planning is accepted only when both fuel margin and object lifetime are feasible.
If planning fails, the agent refreshes obstacle knowledge and retries up to \(k=3\) before switching targets.

\paragraph{Communications}
AgentEnge uses G7P2 target-lock messages to coordinate pickup/fill tasks.
Locks are renewed with bounded TTL and inverse-distance priority; higher-priority locks pre-empt local targets to reduce duplicated trips.

\paragraph{Memory \& Obstacle Handling}
Two-position-history horizons suppress oscillation and short cycles; when a cycle is detected, exploration cursor advancement forces escape behavior.
Fuel-station observations are explicitly maintained to keep survival planning reliable.

\paragraph{Exploration}
The agent uses lawnmower sweeps with strip width \(2r+1\), per-agent phase offsets, and zone-aware bootstrapping.
After coordination stabilizes, waypoint selection is nudged away from teammate-dense areas to reduce sensing overlap.

\subsection{AgentXiong - The Explorer}

AgentXiong is built to maximize team visibility in early and mid stages by prioritizing edge-oriented exploration and broadcasting discovered fuel stations, tiles, and holes.

\paragraph{Core Design Principles}
Its design follows three principles: safe exploration, fuel-bounded behavior, and collaborative information sharing under the common team protocol.

\paragraph{Think Principles}
Decision making follows a hierarchy:
if no fuel station is known, run serpentine search to bootstrap survival.
If \(fuel<150\), refuel immediately; if \(150 \le fuel < 200\), only execute short-range actions that keep the agent near station access.
When fuel is safe and no urgent interaction exists, edge exploration is preferred to expand unknown-area coverage quickly.

\paragraph{Exploration strategy}
The exploration policy biases motion along boundaries and edge structures, which improves frontier discovery versus unconstrained wandering.
Navigation uses a staged fallback: constrained A* first, then greedy move selection, and short-depth BFS as the final recovery option.


\subsection{AgentSun - The Adaptation Specialist}
AgentSun is an adaptation-focused agent designed to remain stable across different map and object configurations without manual retuning.

\paragraph{Planning}
Its decision cascade is: (1) fuel safety, (2) immediate local action (pick/place on current cell), (3) locked task completion, (4) zone sweep exploration.
A lock is kept until target disappearance, unsafe fuel state, or teammate pre-emption.

\paragraph{Adaptive Memory Expiry}
A two-layer memory policy maintains model freshness:
within sensor range, empty observations immediately clear stale entries; beyond range, expiry follows
\[
\text{EXPIRY} = \max\!\big(r,\; 0.7L - \sqrt{A/N}\big),
\]
where \(L\) is object lifetime, \(A\) map area, and \(N\) estimated alive objects.

\paragraph{Three-Tier Fuel Management}
Fuel control combines on-station refuel, critical return protection when margin is tight, and opportunistic refuel when passing near station range (15 cells), with detour-safety checks before tile pursuit.

\paragraph{Exploration}
Exploration uses boustrophedon sweeps with strip spacing \(2r+1\) for systematic coverage.
If blocked, the fallback chain is A* \(\rightarrow\) greedy \(\rightarrow\) random escape.


\subsection{AgentDeng - Utility-Driven Agent}

\paragraph{Decision Architecture} 
AgentDeng uses a utility-driven selector rather than a fixed-priority state machine.
At each step, four candidates (refuel, hole, tile, explore) are scored and the highest utility is executed:
\begin{align*}
  U_{\text{refuel}}  &= 200 \cdot e^{-4m/60}/(1+d), \\
  U_{\text{hole}}    &= 50f/(1+d), \\
  U_{\text{tile}}    &= 30f/(1+d), \\
  U_{\text{explore}} &= 8/(1+d),
  \end{align*}
where \(f = t_{\text{left}}/(t_{\text{left}}+d)\) and \(m\) is fuel margin.
The exponential refuel term balances early over-refuel and late starvation.
To avoid oscillation, replanning is triggered only if the target vanishes or a new option exceeds current utility by \(1.3\times\).

\paragraph{Memory and Exploration}
Memory freshness filtering removes candidates unlikely to survive until arrival.
A visited map guides exploration toward nearest unseen regions, and bootstrap behavior uses zone-restricted snake sweeps to find fuel stations quickly.

\paragraph{Anti-Stuck Fallback}
Path failures are handled by a three-stage fallback:
temporary target blacklist (20 steps), greedy one-step recovery, then memory reset and replanning after persistent stagnation.

\subsection{AgentWu - Verification and Retry Mechanism}
% Describe objective, local policy, planner, memory, strengths/weaknesses.
AgentWu centers on verification-driven planning: cache a path, periodically validate target availability, and retry before full replanning.

\paragraph{Planning} 
The planner combines dynamic target selection, conflict-aware locking, and proactive adaptation.
After locking a target, it plans and caches a path with constrained A*.
If planning fails, the agent performs greedy recovery and retries; after three failed A*+greedy cycles it releases the target, selects a new one, and replans.
A target validity check runs every 15 steps and triggers immediate replanning when a target disappears or is claimed by teammates.
Without an active target, the agent uses zig-zag exploration and directional fallback to keep moving.

\paragraph{Communication} AgentWu follows G7P2, participates in zone coordination, and broadcasts discovered state for team memory sharing.

\paragraph{Fuel management} If \(fuel<150\), refuel has immediate priority to avoid deadlock from fuel depletion.

\paragraph{Utility} A distance-based utility preference (Manhattan distance) biases selection toward closer feasible targets.


\section{Agent Communication \& Collaboration}

Communication is implemented in \texttt{Group7AgentBase} under our latest protocol version \texttt{G7P2}, 
with message tuple \(\langle version, step, from, to, type, x, y, payload\rangle\). 
In \texttt{TWEnvironment}, \texttt{sense()} and \texttt{communicate()} are scheduled before \texttt{think()} and \texttt{act()}, so messages produced in a step are readable by teammates in that same decision cycle.

The protocol combines sparse events and step-wise snapshots. 
Event messages broadcast newly observed objects (\texttt{NT/NH/OB}), one-time fuel-station discovery (\texttt{FS}), and action confirmations (\texttt{PK/FH}). Snapshot message \texttt{SS} encodes local empty/occupied cells each step to refresh shared belief quickly. 
Collaboration is staged: bootstrap zone assignment uses \texttt{ZA/ZK} (deterministic manager election with bounded retries), while runtime conflict avoidance uses \texttt{TL/TR} target locks with priority and TTL. 
Together, these mechanisms reduce redundant pursuit and improve task allocation stability under partial observability.
And the use of target management enpowers our agents to form a expanding area of exploration and exploitation, illustrating as the figure below.

\section{Ablation Study}

We investigated the contribution on the core designs of our agent system by disabling each of them and test on each configuration for 10 runs and take the mean value for comparison.
One is the communication mechanism, we added a \texttt{CommunicationBlackhole} that absorbs all outgoing messages to remove communication layer while keeping other parts of the system unchanged.
The other core design is each of our Agent's local policy and planner, we implemented a \texttt{MinimalAgent} that only has the basic mobility and survival functions provided by the \texttt{Group7AgentBase}, 
with a simple greedy policy to pick up the nearest tile and fill the nearest hole, without any of the additional logic in the original design.

Then we ran the ablation experiments with the settings and the results are shown in the table below.

Ablation reveals the source of improvement. Removing communication reduces reward by \(63.0\%\), \(73.4\%\), and \(97.5\%\) for Sets 1--3, respectively, indicating that shared situational awareness is critical in partially observable and highly dynamic settings. Replacing specialized policies with \texttt{AgentMinimal} also causes substantial drops (\(57.7\%\), \(46.4\%\), \(77.6\%\)), showing that common infrastructure alone is insufficient. Overall, team performance depends on both protocol-level coordination and agent-level planning specialization.

\section{Conclusion}

This project developed a six-agent Tileworld system with a shared engineering backbone (\texttt{Group7AgentBase}) and agent-specific decision modules.
The framework unifies path planning, fuel-safe control, and structured communication (G7P2), while preserving policy diversity across agents.
Empirical results show that the full system consistently outperforms both ablated variants across all tested parameter sets.
Ablation further confirms that communication and policy specialization are complementary; removing either causes major reward loss.
Limitations remain: richer planners can increase runtime overhead on large maps, and heavy reliance on SS messages raises bandwidth cost.
Future work should explore adaptive communication budgets and tighter coupling between global coordination and local plan repair under extreme dynamism.

\section{AgentHanny - MLE for Hazard Estimation Derivation}
Object survival probability modelled as exponential decay:
\[
p_{alive} = e^{-\lambda \Delta t} \quad \in (0,1]
\]

Here, \(\Delta t\) is the estimated time of arrival from the current time stamp, and \(\lambda\) is a reflecting environment \(hazard\) that affects the agent's expectation of an object still alive. 
This \(hazard\) will be different across different experiment setups, such as world size and object creation distribution.

We model each observation on an object as a Bernoulli trial with \(p_{alive}\),
then we can use the maximum likelihood estimation (MLE) to find the best \(\lambda\) that fits the observed data.
By computing the gradient of the negative log-likelihood \(l(P)\), we can then perform the MLE via mini-batch stochastic gradient decent (SGD) with update rule as follows:

\[
\begin{aligned}
\text{Upon Match} \quad
\frac{\partial \ell}{\partial \lambda}
&= -\Delta t
\quad \Longrightarrow \quad
\lambda \leftarrow \max\big(0, \lambda - \eta \Delta t \big)
\\[10pt]
\text{Upon MisMatch} \quad
\frac{\partial \ell}{\partial \lambda}
&= \frac{\Delta t\, e^{-\lambda \Delta t}}{1 - e^{-\lambda \Delta t}}
= \frac{\Delta t}{e^{\lambda \Delta t} - 1} \quad \Longrightarrow \quad
\lambda \leftarrow \lambda + \eta \cdot
\frac{\Delta t}{e^{\lambda \Delta t} - 1 + \epsilon}
\end{aligned}
\]

By using learning rate schedule and gradient ceiling and clipping, the $\lambda$ converges to a reasonable range after 1000 steps on online training, as the curves shown in the figure below.
